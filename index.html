La Historia de HTML

HTML, siglas en inglés de HyperText Markup Language (‘lenguaje de marcado de hipertexto’), hace referencia al lenguaje de marcado para la elaboración de páginas web. Es un estándar que sirve de referencia del software que conecta con la elaboración de páginas web en sus diferentes versiones, define una estructura básica y un código (denominado código HTML) para la definición de contenido de una página web, como texto, imágenes, videos, juegos, entre otros. Es un estándar a cargo del World Wide Web Consortium (W3C) o Consorcio WWW, organización dedicada a la estandarización de casi todas las tecnologías ligadas a la web, sobre todo en lo referente a su escritura e interpretación. HTML se considera el lenguaje web más importante siendo su invención crucial en la aparición, desarrollo y expansión de la World Wide Web (WWW). Es el estándar que se ha impuesto en la visualización de páginas web y es el que todos los navegadores actuales han adoptado.

Filosifia

El lenguaje HTML basa su filosofía de desarrollo en la diferenciación. Para añadir un elemento externo a la página (imagen, vídeo, script, entre otros.), este no se incrusta directamente en el código de la página, sino que se hace una referencia a la ubicación de dicho elemento mediante texto. De este modo, la página web contiene solamente texto mientras que recae en el navegador web (interpretador del código) la tarea de unir todos los elementos y visualizar la página final. Al ser un estándar, HTML busca ser un lenguaje que permita que cualquier página web escrita en una determinada versión, pueda ser interpretada de la misma forma (estándar) por cualquier navegador web actualizado.

Estructura del Documento

Una página web está formada por una serie de páginas enlazadas entre sí, es decir, vamos a tener una serie de ficheros y cada uno de esos ficheros va a contener texto, imágenes, etc. Así que lo primero que deberíamos hacer es un boceto en papel con todo lo que va a contener el documento. Una vez claro este contenido hay que pensar en la estructura de la página y en la organización de los ficheros.


Normalmente hay una página principal, que se suele llamar index.html, con enlaces a otras páginas, que tendrán a su vez otros enlaces y así sucesivamente. Es importante que todo esté bien organizado tanto para que a un usuario le resulte cómodo navegar por ella, como para que al autor le sea fácil poder modificarla o añadir información.


Un documento HTML está formado por tres partes:


  1. Una línea que contiene información sobre la versión de HTML (no siempre),
  2. Una cabecera (delimitada por el elemento HEAD), y
  3. Un cuerpo, con el contenido del documento (delimitado por el elemento BODY)

Todo el documento tiene que ir entre las etiquetas <HTML> </HTML>


El elemento <HEAD></HEAD> contiene información sobre el documento actual, como el título,palabras clave que utilizan los motores de búsqueda, y otros datos que no se consideran parte del contenido de ldocumento porque no se visualizan en el navegador. Sin embargo, pueden poner esta información a disposición de los usuarios a través de otros mecanismos.

Todos los documentos HTML deben tener un elemento <TITLE></TITLE> en la sección HEAD. Los autores deberían utilizar el elemento TITLE para identificar los contenidos de un documento, con lo que se aconseja poner títulos que de verdad reflejen con claridad el contenido del documento. Este título puede contener caracteres, pero no trozos de código.

Atributo href

La etiqueta <a> define un hipervinculo. El atributo href especifica la URL de la pagina a la que va el enlace

El atributo src

La etiqueta <img> se utiliza para incrustar una imagen en una pagina HTML. El src el atributo especifica la ruta a la imagen que se mostrara:

<img src="img/image.jpg">

Hay dos formas de especificar la URL en el atributo src

  1. URL absoluta : enlaces a una imagen externa alojada en otro sitio web. Ejemplo: src ="https://fonts.google.com/specimen/Roboto"

  2. URL relativa: enlaces a una imagen alojada en el sitio web. Aquí, la URL no incluye el nombre de dominio. Si la URL comienza sin una barra, será relativa a la página actual. Ejemplo: src ="img_tarea.jpg".Si la URL comienza con una barra, será relativa al dominio. Ejemplo: src = "/ images / img_tarea.jpg".

Sugerencia: Casi siempre es mejor usar las URL relativas. No se romperan al cambiar de dominio.

El atributo alt

El atributo alt requerido para la etiqueta <img>. Especifica un texto alternativo Esto sirve cuando el usuario tiene una conexión lenta o hay un problema con el atributo src. Ejemplo:

<img src="mouse.jpg" alt="La imagen de un raton de computadora">

El atributo lang

Siempre debe incluir el atributo lang dentro de la etiqueta <html>, para declarar el idioma de la página web. Esto está destinado a ayudar a los motores de búsqueda y navegadores.


El siguiente ejemplo especifica el inglés como idioma:

<!DOCTYPE html> <html lang="en">
<head>
<title></title>
</head>
<body>
</body>
</html>

Los códigos de país también se pueden agregar al código de idioma en el atributo lang. Entonces, los dos primeros caracteres definen el idioma de la página HTML y los dos últimos caracteres definen el país.

El siguiente ejemplo especifica inglés como idioma y Estados Unidos como país:

<!DOCTYPE html> <html lang="en-US">
<head>
<title></title>
</head>
<body>
</body>
</html>

Sugerencias

En HTML no se distingue de mayúsculas o minúsculas pero lo más recomendable es escribirlo en minúsculas ya que casi s iempre el código estará en minúsculas (en mi experiencia no me encontrado con ninguna página web que tenga las etiquetas o atributos en mayúsculas),

Una Duda que puede surgir es si usar comillas dobles o simples y la respuesta es, que puedes usar las dos. Claro si un atributo tiene comillas Entonces se tienen que usar las dos ,aqui un ejemplo.

Formatos de texto en HTML

HTML contiene varios elementos para definir texto con un significado espacial. Aqui una lista, y despues vamos a repasar cada uno y que significan.

Elementos de formato HTML

  • <b> - Texto en negrita
  • <strong> - Texto importante
  • <i> - Texto en cursiva
  • <em> - Texto enfatizado
  • <mark> - Texto marcado
  • <del> - Texto eliminado
  • <ins> - Texto insertado
  • <sub> - Texto de subindice
  • <sup> - Texto de superindice
  • <small> - Texto más pequeño

HTML <b> y <strong> elementos

El elemento <b> HTML define texto en negrita, sin ninguna importancia adicional.

<b> Este parrafo esta en negrita</b>

El elemento <strong> define el texto con gran importancia. El contenido del interior se muestra normalmente en negrita.

<strong> Este texto es mas importante </strong>

Elementos HTML <i> y <em>

El elemento <i> define una parte del texto en una voz o estado de ánimo alternativo. El contenido del interior se muestra normalmente en cursiva.

Consejo: la etiqueta <i> se utiliza a menudo para indicar un término técnico, una frase de otro idioma, un pensamiento, el nombre de un barco, etc.

<i>Este es un texto en italica</i>
Este es un texto en italica

La etiqueta <em> define texto enfatizado. El contenido del interior se muestra normalmente en cursiva.

Consejo: un lector de pantalla pronunciara las palabras <i> con énfasis, utilizando el estrés verbal.

<em>Este es un texto con énfasis.</em>
Este es un texto con énfasis.

Elemento <small>

El elemento <small> define un un texto mas pequeño de lo normal aqui un ejemplo:

<small>Este es un texto muy pequeño.</small>
Este es un texto muy pequeño.

Elemento <del>

El elemento <del>define el texto que se ha eliminado de un documento. Los navegadores generalmente tacharán una línea a través del texto eliminado:

<p>Este es un texto esta <del>tachado.</del> </p>

Elemento <ins>

El elemento <ins> define un texto que se ha insertado en un documento. Los navegadores suelen subrayar el texto insertado:

<p>Mi color favorito es <del>rojo</del> <ins>azul</ins> </p>

Elemento <sub>

El elemento <sub> El texto en subíndice aparece medio carácter debajo de la línea normal y, a veces, se representa en una fuente más pequeña. El texto de subíndice se puede utilizar para fórmulas químicas, como H 2 O:

<p>Este es un texto <sub>subindice.</sub> </p>

Este es un texto subindice.

Elemento <sup>

El elemento <sup> define texto en superíndice. El texto en superíndice aparece medio carácter por encima de la línea normal y, a veces, se representa con una fuente más pequeña. El texto en superíndice se puede utilizar para notas al pie, como WWW [1]:

<p>Este es un texto <sup>superindice.</sup> </p>

Este es un texto superindice.
